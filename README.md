# Генератор процедурного ландшафта (ВКР)

Приложение на **Python 3.11–3.12** (рекомендуется; 3.14 допустим при рабочем PySide6): клеточный автомат **Day & Night** для сушеобразования и береговой линии, модель высот **`tapered_height`**, десктоп-интерфейс (**Qt 6** + Matplotlib), экспорт **OBJ + MTL** для Blender и серии экспериментов (CSV).

---

## 1. Что нужно установить до запуска

- **Python** 3.11 или новее (`python3 --version`).
- Зависимости из **`requirements.txt`**: NumPy, Matplotlib, SciPy (опционально ускоряет сглаживание), **PySide6**.

Pygame нужен только для режима **`lab`** (отдельная лаборатория CA по консоли) — в основную работу ВКР не входит.

---

## 2. Установка (один раз)

Откройте терминал в папке проекта (**`project generate`**, здесь лежит `requirements.txt`, `run.sh`, пакет `vkr_terrain/`).

### Windows

```bat
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

---

## 3. Запуск основного приложения (то, что сдаётся по ВКР)

Рабочая директория должна быть **корнем проекта** (чтобы нашлись `vkr_terrain` и `output/`).

| Способ | Команда |
|--------|---------|
| **macOS** | Двойной щелчок по **`run.command`** (откроется Terminal) или вручную: `./run.command` |
| **Linux/macOS** | `./run.sh` или `bash run.sh` |
| **Windows** | **`run.bat`** или в CMD/PowerShell из папки проекта после активации venv |

Универсально (после `activate`):

```bash
python -m vkr_terrain
```

или явно:

```bash
python -m vkr_terrain.desktop_app
```

Окно: три вкладки **«Генерация»**, **«Эксперименты»**, **«Справка»** и большая область предпросмотра справа.

---

## 4. Кратко по интерфейсу

### Вкладка «Генерация»

- Задайте размер сетки, вероятность суши **p_land**, сид, тор (чекбокс), итерации берега, слои и итерации высот и т. д.
- **«Сгенерировать»** (или **Ctrl+G**) — пайплайн в фоне, справа обновится доска этапов.
- **Сохранить PNG** (**Ctrl+S**) — сохранение картинки текущего предпросмотра.
- **Экспорт OBJ для Blender** (**Ctrl+Shift+O**) — только **после успешной генерации**. В файле: суша (как в предпросмотре по XY: колонка → **X**, строка → **Y**, без зеркала относительно 2D-карты) и **`water_surface`** — горизонт чуть ниже **z = 0** (тот же нуль, что после привязки рельефа в приложении).

Импорт в Blender: **File → Import → Wavefront (.obj)**; при «перевёрнутой» карте проверьте в импорте оси (**Forward / Up**) или поворот объекта целиком на 180° по Z. Во **Viewport Shading → Material Preview** видны материалы **Land** и **Sea**.

### Вкладка «Эксперименты»

1. По желанию нажать **«Запустить эксперименты ЛР2–ЛР4»** (или меню **Файл → Запустить эксперименты…**) — счёт займёт время, результаты в **`output/research/`** (CSV).
2. **«Обновить из CSV»** подгружает таблицы и строит **кандидатов** топ-5 (набор параметров, не «картинка» до генерации).
3. **«Применить и показать результат»** — параметры из выбранного кандидата в форму и запуск генерации.
4. **«Подтвердить выбор вручную (JSON)»** — сохранить зафиксированный выбор для отчёта.

### Файл

- Импорт / экспорт **JSON параметров** — для воспроизведения той же карты после загрузки и **«Сгенерировать»**.
- Прочие режимы (см. ниже).

### Вкладка «Справка» в программе

Дублирует и дополняет этот файл в сжатом виде для быстрого доступа из приложения.

## 5. Дополнительные режимы из консоли

Из папки проекта с активированным venv:

| Команда | Назначение |
|---------|------------|
| `python -m vkr_terrain.main research` | Только эксперименты в CSV, без окна GUI |
| `python -m vkr_terrain.main --quick` | Разовая генерация, PNG **`output/generation_full.png`** |
| `python -m vkr_terrain.main gui` | То же главное приложение Qt |
| `python -m vkr_terrain.main lab` | Нужен **pygame**; лаборатория CA |

## 6. Структура каталогов (что отправляют)

```
project generate/
├── README.md                 ← этот файл (инструкция)
├── requirements.txt
├── run.sh / run.bat / run.command
├── vkr_terrain/              ← исходный код пакета
└── output/                   ← создаётся при работе (PNG, OBJ, research/…); в комплекте поставки можно оставить пустым
```

Папка **`.venv`** при передаче преподавателю **не обязательна**: достаточно исходников и инструкции по созданию окружения (раздел 2).

---

## 7. Проверка перед отправкой

1. Свежее venv, `pip install -r requirements.txt`.
2. `python -m vkr_terrain` — окно открывается, генерация выполняется.
3. После генерации экспорт OBJ, импорт в Blender без критических артефактов (проверено локально условно).

Удачи с защитой ВКР.
